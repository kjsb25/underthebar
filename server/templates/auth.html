Hevy

Status: {{ "authorized" + (' as ' + hevy_username if hevy_username else '') if hevy_authorized else "not authorized" }}

Hevy has no public OAuth flow, so you have to bootstrap by handing this app a session you've already created in your browser. Easiest path: copy the auth2.0-token cookie out of a logged-in hevy.com tab and paste it below.

Step-by-step: get the cookie from your browser
  1. Open hevy.com in a desktop browser and sign in.
  2. Open DevTools:
    • Chrome / Edge / Brave: F12 or Ctrl/Cmd+Shift+I, then Application tab → Storage → Cookies → https://hevy.com.
    • Firefox: F12, then Storage tab → Cookies → https://hevy.com.
    • Safari: enable Develop menu (Settings → Advanced → Show features for web developers), then Develop → Show Web Inspector → Storage → Cookies.
  3. Find the row named auth2.0-token. The value looks like %7B%22access_token%22%3A%22… (URL-encoded JSON).
  4. Click the value to select it, copy it, and paste it into the Cookie value field below. We'll decode it for you.
  5. Hit Save tokens. If it worked, the status pill above will flip to "authorized as <your name>".

The cookie is bearer credentials — anyone with it can act as you on Hevy until it expires. Don't share it, and clear it from any shared machine when you're done.

Advanced: paste tokens individually

Use this if you already have the decoded tokens — e.g. from ~/.underthebar/session.json on a machine that's run the desktop app, or because you've decoded the cookie yourself. Leave the cookie value field above blank when using these.
